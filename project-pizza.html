// Overview

The Challenge

A pizza shop needed to understand its sales performance across a full year of transaction data: 48,620 rows covering every order, pizza type, size, category, and timestamp. The goal was to build automated SQL queries for 12+ KPIs and deliver an interactive dashboard that would replace hours of manual reporting for stakeholders.

// Dashboard

Tableau Dashboard

Interactive Tableau dashboard surfacing 12+ KPIs across revenue, volume, timing, and category performance

// SQL

Automated Computations

20+ SQL queries were written in MySQL to compute all KPIs, trend analyses, and ranking breakdowns: automating what previously required hours of manual spreadsheet work.

Revenue & averages
-- Total revenue SELECT SUM(total_price) FROM Pizza_sales; -- Avg order value SELECT ROUND(SUM(total_price) / COUNT(DISTINCT order_id), 2) FROM Pizza_sales;
Trend analysis
-- Hourly trend for pizzas sold SELECT HOUR(order_time) AS order_hour, SUM(quantity) AS total_pizzas_sold FROM Pizza_sales GROUP BY HOUR(order_time) ORDER BY HOUR(order_time);

// Insights

Key Findings

Peak Hours
Lunch rush (12 to 1 PM) and dinner (5 to 7 PM) drive the highest volume. Orders drop sharply after 9 PM.
Size Preference
Large (45.9%) dominates, followed by Medium (30.5%) and Regular (21.8%). XL and XXL under 2% combined.
Balanced Categories
All 4 categories within 3% of each other. Classic leads at 26.9%, showing broad menu appeal across the board.
