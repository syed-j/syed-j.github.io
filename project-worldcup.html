// Overview

The Challenge

With the 2026 FIFA World Cup expanding to 48 teams for the first time, hosted across the USA, Mexico, and Canada, the question was simple: can machine learning predict who wins using nothing but historical match data?

This was built entirely from scratch with zero prior ML experience: from raw CSV files to a trained model running 10,000 simulated tournaments. Every decision, from data verification to model selection, was driven by the data itself.

// Data foundation

Finding Real Data in a Sea of Fake

Not all datasets are what they claim to be. Four sources were evaluated before settling on the right combination. One dataset (832 matches with zero missing values) looked perfect until the finals were verified.

Algeria beat Argentina in the 2022 final. Iran won the 1990 World Cup. "Draw" won in 1982. The entire dataset was AI-generated fiction. It was discarded entirely. Lesson: always verify your data before building anything on top of it.

// Process

From Raw Data to Predictions

Phase 1
Environment
Python Setup in VS Code
Configured a Python 3.13 virtual environment in VS Code with pandas, numpy, scikit-learn, XGBoost, matplotlib, seaborn, and Jupyter. Fixed kernel detection issues and an OpenMP runtime error before a single line of ML code was written.
Phase 2
Data
Data Collection & Verification
Collected 4 datasets, verified each against known World Cup results, discarded one fake dataset, and merged the remaining real sources into a single master file covering 864 matches from 1930 to 2022.
Phase 3
Features
Feature Engineering
Built team strength features from historical data: win percentage, average goals scored, average goals conceded, and average goal difference. Calculated using a sliding window so the model never sees the future. For each match, the model sees the difference in these stats between the two teams at the time they played.
Phase 4
Modelling
Model Training & Selection
Trained three models on 563 historical matches. First attempt predicted exact winners by name (30+ classes, 29% accuracy). Reframed the problem to 3 outcomes (home win, away win, draw) and accuracy jumped to 54-56%. Random Forest was selected over Logistic Regression despite a slightly lower score: it actually predicted all three outcomes rather than defaulting to home win every time.
Phase 5
Simulation
10,000 Tournament Simulations
Built a tournament simulator that shuffles 48 teams, plays every match using model predictions, and records the winner. Initial version took 20+ minutes due to recalculating team features inside every match prediction. Solved with a pre-computed matchup cache: all 2,256 possible matchups calculated once, then looked up instantly. Runtime dropped to under 3 seconds.
Phase 6
Results
Predictions & Visualisation
Compiled win probabilities, final reach rates, and semi-final rates for all 48 teams across 10,000 simulations. Visualised results as a professional grouped bar chart.
// Models

Choosing the Right Model

Three models were trained and evaluated on the same 80/20 train-test split. The winner wasn't the most accurate: it was the most honest.

Logistic Regression
56.6%
Highest raw accuracy but achieved it lazily: predicted home win almost every time and never once predicted a draw. Useful as a baseline but not trustworthy for simulation.
Random Forest ✓
54.0%
Slightly lower accuracy but predicted all three outcomes meaningfully. Better F1 scores on away wins and draws. The right model for simulating a realistic tournament where any result is possible.
XGBoost
47.8%
Came in third on this dataset. XGBoost often outperforms Random Forest on larger tabular datasets. With more data and feature engineering it would likely improve significantly.

// Learnings

What This Project Taught

Garbage In, Garbage Out
A model trained on fake data produces fake predictions. Data verification isn't optional: it's the most important step in any ML project. Always sanity-check your data against known ground truth before building anything.
Problem Framing Matters More Than the Algorithm
Predicting 30+ team names got 29% accuracy. Reframing to 3 outcomes (home win, away win, draw) got 54%. Same data, same model. The single biggest accuracy gain came from thinking differently about the problem, not tuning hyperparameters.
Accuracy Isn't Everything
Logistic Regression scored higher but never predicted a draw. It was gaming the metric. Random Forest scored lower but was honest about uncertainty. For a simulation tool, honesty beats raw accuracy.
Cache What You Can
The simulator took 20+ minutes because it recalculated team features inside every match prediction: 1.4 million redundant calculations. Pre-computing all 2,256 matchups once dropped runtime to under 3 seconds. Profile before optimising.
